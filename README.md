# motionanalyzer
time series motion analyzer

cursor IDE 환경에서 바이브코딩을 통한 장기 물리학 패턴 분석 그래프 프로젝트
미리 바이브에 최적화된 MCP 및 디렉토리 환경구축
아래 청사진 내용 구현에 최적화된 사전준비 및 파일 구조, MCP, cursor 파일 등을 먼저 작성하고 프로젝트를 시작해라.

Input 1 : 하나의 비디오 파일에서 추출된 시간순서로 연속적으로 처음부터 끝까지 정렬된 프레임에서 중요한 테두리를 점좌표화 시킨 txt 파일들의 묶음, 파일명으로 순서가 구분됨.
*각 txt 파일의 포멧

# x,y,index
1097,1087,1
1096,1086,2
1095,1086,3
1094,1086,4
1094,1086,5
1093,1085,6
1092,1085,7
1091,1085,8
1090,1084,9
1089,1084,10
1088,1084,11
1087,1083,12
1087,1083,13
1086,1083,14
1085,1082,15
1084,1082,16
1083,1081,17
1082,1081,18

txt 파일 하나당 약 200~300개 정도의 라인으로 이루어져 있음

Input 2 : 지정된 txt 파일 묶음의 초당 프레임 수 fps(속도 및 가속도 벡터 분석을 위해 사용됨) 

Output 1 : 이미지 묶음에서 같은 index끼리의 프레임별 x,y 좌표 각각의 이동경로 벡터, 속도 벡터, 가속도 벡터를 표준화된 방식으로 정리한 txt 파일
Output 2 : output 1에서 출력된 결과를 시각화 및 분석하여 패턴을 사용자가 이해하기 쉽도록 하고, 다른 비디오를 분석해서 나온 결과물을 업로드 하여 동시분석을 통해 차이점과 특이점을 찾기 쉽도록 도와주는 gui 기반 분석화면, 이미지 파일 추출기능포함
