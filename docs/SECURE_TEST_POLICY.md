# Secure Test Policy

## 목적

실제 사내 데이터는 외부 반출이 불가능하므로, 개발/검증은 두 단계로 분리한다.

1. 외부 개발 환경: 가상(합성) 데이터 기반 기능 검증
2. 사내 보안 환경: exe 기반 실제 데이터 평가

## 원칙

- 실제 원본 데이터, 원본 이미지, 좌표 원문은 GitHub에 업로드하지 않는다.
- 외부 환경에서는 물리학 기반 합성 데이터를 사용해 재현 가능한 테스트를 수행한다.
- 사내 환경에서는 GitHub에서 내려받은 exe만 사용해 실제 데이터를 평가한다.
- 외부로 공유 가능한 산출물은 비식별 로그/지표/요약 결과로 제한한다.

## 운영 절차

### 1) 외부 개발 환경 (GitHub 연결 가능 환경)

1. 코드 변경 후 합성 데이터 테스트 실행
   - `motionanalyzer gen-synthetic`
   - `pytest`
2. 변경 사항 커밋/푸시
3. GitHub Actions `build-windows-exe`에서 exe 생성
4. 생성된 아티팩트(`motionanalyzer-cli.exe`) 확인

### 2) 사내 보안 환경 (실데이터 환경)

1. GitHub 아티팩트에서 exe 다운로드
2. 사내 PC의 실제 데이터 경로에 배치하여 실행
3. 실데이터 결과 로그 생성
4. 로그에서 민감정보 제거 후 외부 공유

## 로그 공유 규칙

- 공유 가능
  - 처리 시간(ms), 프레임 수, 포인트 수
  - 계산 성공/실패 통계
  - 비식별 에러 메시지
- 공유 금지
  - 원본 좌표 전체
  - 원본 프레임/이미지
  - 사내 경로/시스템 식별 정보

## 성공 기준

1. 합성 데이터 테스트 통과
2. 사내 실데이터 실행 성공
3. 비식별 로그 기준으로 품질 지표 달성
